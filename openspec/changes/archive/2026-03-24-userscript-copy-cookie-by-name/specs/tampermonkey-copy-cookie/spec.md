## 新增需求

### 需求:用户必须能够配置要复制的 Cookie 名称

用户脚本必须允许用户指定一个 Cookie 名称（字符串），并在后续「复制」操作中使用该名称从当前页面的 `document.cookie` 中查找对应项。名称必须通过用户可访问的配置方式设置或修改（例如油猴菜单与存储 API），禁止要求用户仅能通过修改脚本源码来更改名称。

#### 场景:通过菜单修改名称

- **当** 用户选择脚本提供的「设置 Cookie 名」类菜单项并输入非空名称
- **那么** 脚本必须持久保存该名称，并在之后的复制逻辑中使用该名称

#### 场景:名称未配置时的提示

- **当** 用户触发复制操作但尚未配置有效 Cookie 名称
- **那么** 脚本必须提示用户先配置名称，且不得将空内容作为成功复制处理

### 需求:脚本必须复制当前页可读 Cookie 中匹配名称的值

在已配置 Cookie 名称的前提下，脚本必须从 `document.cookie` 中解析键值对；若存在与配置名称匹配的项，则必须将该 Cookie 的**值**（value）写入系统剪贴板或提供用户可一键复制的等价结果。

#### 场景:成功复制

- **当** `document.cookie` 中存在与配置名称匹配的 Cookie 且值非空
- **那么** 剪贴板内容必须为该 Cookie 解码后的值（或与管理器一致的展示形式）

#### 场景:不存在或 HttpOnly

- **当** 当前页 `document.cookie` 中不存在该名称（包括该 Cookie 仅为 HttpOnly 因而不可见）
- **那么** 脚本必须给出明确失败或说明性提示，且不得声称已成功复制

### 需求:脚本必须说明 HttpOnly 限制

脚本或其配套说明中必须包含表述：无法通过页面脚本读取 HttpOnly Cookie，本工具仅适用于可通过 `document.cookie` 访问的 Cookie。

#### 场景:用户阅读脚本头部或文档

- **当** 用户查看脚本的元数据注释或仓库内简短说明
- **那么** 必须能看到 HttpOnly 不可读的说明
